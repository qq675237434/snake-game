## 已知问题和限制

### 当前版本限制
1. **特效简化**：
   - 火焰特效使用简单的粒子动画
   - 发光特效使用 Canvas shadowBlur，性能开销较大
   - 彩虹效果仅改变颜色，未添加渐变过渡

2. **性能考虑**：
   - 长蛇身（>50 节）时，彩虹效果计算可能影响帧率
   - 发光特效在低端设备上可能卡顿

3. **解锁系统**：
   - 成就检测仅在游戏结束时进行
   - 无法手动重置解锁进度（需清除浏览器数据）

4. **UI/UX**：
   - 皮肤选择界面无搜索/筛选功能
   - 皮肤描述较短，无详细说明弹窗
   - 预览界面静态，未展示动态特效

### 浏览器兼容性
- ✅ Chrome/Edge: 完全支持
- ✅ Firefox: 完全支持
- ✅ Safari: 完全支持
- ⚠️ 旧版浏览器（IE）：不支持

### 本地存储限制
- 使用 localStorage，数据仅在当前浏览器有效
- 存储容量约 5MB，足够存储皮肤数据
- 隐私模式下数据可能不保存

## 性能优化建议

### 已实现优化
1. 避免在渲染循环中创建新对象
2. 使用 requestAnimationFrame 的替代方案（固定帧率）
3. 特效计算使用简单的三角函数
4. 颜色插值结果可缓存（未实现）

### 未来优化方向
1. 使用 WebGL 替代 Canvas 2D（大幅提升性能）
2. 预渲染皮肤贴图为 Sprite
3. 使用对象池管理特效粒子
4. 降级特效质量（低性能设备）

## 调试技巧

1. **查看已解锁皮肤**：
   ```javascript
   console.log(game.skinManager.unlockedSkins);
   ```

2. **手动解锁所有皮肤**：
   ```javascript
   Object.keys(SKINS).forEach(id => game.skinManager.unlockSkin(id));
   game.skinManager.saveUnlockedSkins();
   location.reload();
   ```

3. **查看当前皮肤**：
   ```javascript
   console.log(game.currentSkin);
   ```

4. **重置所有进度**：
   ```javascript
   localStorage.clear();
   location.reload();
   ```
